# AI Tracker

**AI Tracker** is an automated tool written in Go that monitors your AI API usage through [OpenRouter](https://openrouter.ai/) and syncs detailed reports directly to your [Obsidian](https://obsidian.md/) vault stored in a Git repository (GitHub or Forgejo).

---

## How It Works

The complete process flow is as follows:

1. **Data Extraction**: Connects to the OpenRouter API using management credentials to fetch activity history for the last 30 days.

2. **Grouping**: Activity data is organized in two dimensions:
   - **Daily**: Each day with activity generates an individual file.
   - **Monthly**: Each month accumulates all activities to generate a consolidated report.

3. **AI Analysis**: For each report (daily and monthly), **DeepSeek** is used via the OpenRouter API to generate an executive summary in Spanish that highlights:
   - The most used model.
   - The model with the highest spending.
   - Usage patterns and implicit recommendations.

4. **Report Generation**: Markdown files are rendered with:
   - **YAML Frontmatter** compatible with Obsidian: report type, date, month, total cost, tokens consumed, number of requests, and top models.
   - **AI Summary**: Analytical paragraph generated by DeepSeek.
   - **Breakdown Table**: List of all models used with requests, cost in dollars, request percentage, and cost percentage.

5. **Git Sync**: Each report is automatically pushed to the Git repository configured in the Obsidian vault path (default: `10 - AI Related/OpenRouter`).

## Prerequisites

- **Go 1.25+** installed
- **OpenRouter** account with:
  - **Management Key**: To query activity history.
  - **API Key**: To generate summaries with DeepSeek.
- A **Git repository** (GitHub or Forgejo) that serves as your **Obsidian Vault**.
- A **personal access token** with read/write permissions on the repository.

---

## Configuration

Create a `.env` file in the project root with the following variables:

```bash
# OpenRouter
MANAGEMENT_KEY=sk-or-v1-xxxxx  # Management key for history
API_KEY=sk-or-v1-xxxxx         # API key for chat calls

# GitHub / Forgejo
GIT_TOKEN=ghp_xxxxx            # Personal access token
GIT_OWNER=your-username        # Repository owner
GIT_REPO=my-obsidian-vault    # Repository name

# Optional: Base path in vault (default: "10 - AI Related/OpenRouter")
OBSIDIAN_BASE_PATH=10 - AI Related/OpenRouter
```

## Automation

To run the tracker automatically, you can set up a **cron job** or use **GitHub Actions**:

### Cron Example (Linux)

```bash
0 6 * * * cd /path/to/ai-tracker && ./ai-tracker >> /var/log/ai-tracker.log 2>&1
```

### GitHub Action Example

```yaml
name: Daily AI Tracker Sync
on:
  schedule:
    - cron: '0 6 * * *'  # Everyday at 6 AM
  workflow_dispatch:

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.25'
      - run: go run main.go
        env:
          MANAGEMENT_KEY: ${{ secrets.MANAGEMENT_KEY }}
          API_KEY: ${{ secrets.API_KEY }}
          GIT_TOKEN: ${{ secrets.GIT_TOKEN }}
          GIT_OWNER: ${{ secrets.GIT_OWNER }}
          GIT_REPO: ${{ secrets.GIT_REPO }}
```

---

## Security

- **Never version the `.env` file** — it's already in `.gitignore`.
- Use **Secrets** in GitHub Actions for credentials.
- The Git token should have minimum permissions: `repo` (read/write content).
